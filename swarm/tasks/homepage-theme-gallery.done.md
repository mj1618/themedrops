# Build the homepage theme gallery

## Context
The home page (`app/src/app/page.tsx`) is still showing default Next.js boilerplate ("To get started, edit the page.tsx file" with Deploy/Docs links). The `themes.list` paginated query and `themes.search` query already exist on the backend. The theme detail page at `/theme/[slug]` is fully built. The homepage is the front door of the app and the most critical missing piece for MVP.

The layout (`app/src/app/layout.tsx`) also still has default metadata: title "Create Next App" and description "Generated by create next app" — these need updating too.

## Scope
Replace the placeholder home page with a working theme gallery that lists public themes, lets users click through to detail pages, and includes basic search. Also fix the site metadata. Should be design-forward per the project's goals.

## Tasks

### 1. Fix site metadata in layout.tsx
- **File:** `app/src/app/layout.tsx`
- Change title from "Create Next App" to "ThemeDrops" (or "ThemeDrops - Theme Gallery")
- Change description to something appropriate like "Discover and share color and typography themes for any application"

### 2. Replace homepage with theme gallery
- **File:** `app/src/app/page.tsx`
- Make it a `"use client"` component (needs Convex hooks)
- Use `usePaginatedQuery(api.themes.list, {}, { initialNumItems: 20 })` to fetch themes
- Display a hero/header section with the site name and a short tagline
- Render a responsive grid of theme cards

### 3. Theme card design
Each card should show:
- Theme name
- Color palette preview (the 6 colors as small swatches in a row)
- Star count
- Author name (resolve via `users.get` or include in a joined query)
- Link to `/theme/[slug]`
- The card itself could use the theme's colors as a subtle background/border treatment to make browsing visually interesting

### 4. Search bar
- Add a search input at the top of the gallery
- Use `useQuery(api.themes.search, { name: searchTerm })` when the user types (debounced)
- When search is active, show search results instead of the paginated list
- When search is cleared, return to the default paginated list

### 5. Load more / pagination
- Add a "Load more" button at the bottom that calls `loadMore(20)` from the paginated query
- Show loading state while fetching
- Hide the button when `status === "Exhausted"`

### 6. Empty and loading states
- Show a loading skeleton/spinner while initial data loads
- Show a friendly empty state if no themes exist
- Show "No results" when search returns nothing

## Acceptance criteria
- [x] Home page displays a grid of public theme cards fetched from Convex (not hardcoded)
- [x] Each card shows theme name, color swatches, and star count
- [x] Clicking a card navigates to `/theme/[slug]`
- [x] Search input filters themes by name using the existing search query
- [x] "Load more" button works for pagination
- [x] Loading and empty states are handled
- [x] Site title/metadata says "ThemeDrops", not "Create Next App"
- [x] Page is styled with Tailwind, looks polished and design-forward
- [x] No new npm dependencies required

## Technical notes
- `themes.list` uses `paginationOptsValidator` — use `usePaginatedQuery` from `convex/react`
- `themes.search` takes `{ name: string }` and returns up to 20 public themes
- Theme objects include: name, slug, description, authorId, starCount, colors (bg/fg/primary/secondary/accent/muted), fonts
- The theme detail page at `/theme/[slug]/page.tsx` is already built and can be used as a style reference
- Convex guidelines are in `convex/_generated/ai/guidelines.md` — read before writing Convex code
- Next.js docs are in `node_modules/next/dist/docs/` — check for breaking changes before writing page code
- Tailwind v4 is in use
